# 아임웹 상품 요약설명 도우미 (Chrome 확장)

아임웹 관리자 > 쇼핑 > 상품 수정 화면의 "요약 설명(Meta)" 입력을 돕는 크롬 확장 프로그램(Manifest V3).
**툴바 아이콘을 누르면 팝업**이 열리고, 상품명·카테고리를 자동 반영한 초안을 톤별로 만들어 에디터에 바로 적용한다.
페이지에는 아무것도 상시 주입하지 않는다 — 아이콘을 누른 순간에만 해당 탭에 접근한다(activeTab).

## 사용법

1. 상품 수정 화면을 연다
2. 툴바의 확장 아이콘 클릭 (자주 쓰면 핀 고정 추천)
3. 톤 선택(정중한 / 감성적 / 간결한) → 문구가 마음에 안 들면 [다른 문구]로 로테이션(톤당 3종)
4. 미리보기에서 자유 수정 → [에디터에 적용] → 팝업 자동 닫힘
5. 아임웹 [저장]을 눌러야 실제 저장된다. 적용 취소는 에디터에서 Ctrl+Z

## 기능

- 상품명·카테고리 자동 인식 + 받침에 따른 조사(은/는, 이/가, 을/를, 과/와) 자동 선택
- 톤 3종 x 문형 3종 = 초안 9종, 글자수 표시, 마지막 사용 톤 기억
- 상품 수정 화면이 아니면 안내 문구만 표시

## 설치 (개발자 모드)

1. `chrome://extensions` 접속
2. 우측 상단 "개발자 모드" 켜기
3. "압축해제된 확장 프로그램을 로드합니다" 클릭 → 이 폴더(`imweb-summary-helper`) 선택
4. 툴바 퍼즐 아이콘에서 "요약설명 도우미"를 핀 고정하면 바로 쓸 수 있다

## 동작 원리

- 권한은 `activeTab` + `scripting`뿐 — 아이콘을 누른 탭에만, 누른 순간에만 접근한다. 상시 콘텐츠 스크립트 없음
- 적용은 `chrome.scripting.executeScript({world: "MAIN"})`으로 페이지의 Froala 인스턴스(`window.FroalaEditor.INSTANCES`)를 찾아 `instance.html.set()` + `contentChanged` 트리거로 반영 — 아임웹 우측 미리보기·저장 로직과 그대로 연동된다(요약설명 에디터 = Froala 3.1.1). 인스턴스를 못 찾으면 contenteditable DOM 조작 + `input` 이벤트로 폴백
- 관리자 DOM은 해시된 클래스명이라 셀렉터에 의존하지 않고 **라벨 텍스트 앵커**("요약 설명" / "상품명" / "카테고리")로 탐색한다
- v1은 페이지 주입형 인라인 패널이었으나 화면을 차지하는 문제로 v2에서 팝업형으로 전환 (git 히스토리 참고)

## 파일 구조

- `manifest.json` — MV3 선언 (action popup, activeTab + scripting)
- `popup.html` / `popup.js` — 팝업 UI + 템플릿 + 페이지 주입 함수
- `icons/` — 툴바 아이콘

## 제약

- 요약 설명 필드 전용 (상세 설명 에디터는 대상 아님)
- 적용한 내용은 아임웹 [저장]을 눌러야 실제 저장된다
- 실측 기준: 아임웹 관리자 상품 수정 화면, Froala 3.1.1 (2026-07 확인)
