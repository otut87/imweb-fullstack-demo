# 아임웹 상품 요약설명 도우미 (Chrome 확장)

아임웹 관리자 > 쇼핑 > 상품 편집 화면의 "요약 설명(Meta)" 입력을 돕는 크롬 확장 프로그램(Manifest V3).
요약 설명 에디터 바로 위에 도우미 패널을 주입해, 상품명·카테고리를 자동 반영한 초안을 톤별로 생성하고 한 번에 적용한다.

## 기능

- 톤 3종(정중한 / 감성적 / 간결한) x 문형 3종(핵심 소개 / 감성 어필 / 혜택 강조) = 초안 9종
- 상품명·카테고리 자동 반영 + 받침에 따른 조사(은/는, 이/가, 을/를, 과/와) 자동 선택
- 초안 미리보기에서 자유 수정 후 [에디터에 적용]
- [되돌리기] — 적용 직전 내용으로 복원 (최대 10단계)
- 에디터 글자수 카운터 — 메타 설명 권장 80~160자 기준 색상 표시 (권장 구간 녹색, 초과 적색)

## 설치 (개발자 모드)

1. `chrome://extensions` 접속
2. 우측 상단 "개발자 모드" 켜기
3. "압축해제된 확장 프로그램을 로드합니다" 클릭 → 이 폴더(`imweb-summary-helper`) 선택
4. 아임웹 관리자 > 쇼핑 > 상품 > 상품 수정 화면을 열면 요약 설명 위에 패널이 나타난다

## 동작 원리

- `content.js`가 `world: "MAIN"`(페이지 컨텍스트)으로 실행되어 페이지의 Froala 에디터 인스턴스(`window.FroalaEditor.INSTANCES`)에 직접 접근한다. isolated world에서는 페이지 전역 객체에 닿을 수 없어 MAIN world를 사용한다 (Chrome 111+).
- 적용 시 `instance.html.set()` + `contentChanged` 트리거로 반영하므로 아임웹 우측 미리보기·저장 로직과 그대로 연동된다. 인스턴스를 못 찾는 경우 contenteditable DOM 조작 + `input` 이벤트로 폴백.
- "요약 설명" 라벨을 앵커로 같은 섹션의 에디터만 찾는다 — 상품 상세 설명 에디터를 건드리지 않는다 (관리자 화면의 해시된 클래스명에는 의존하지 않음).
- 관리자가 SPA처럼 동작하거나 에디터가 늦게 렌더되어도 동작하도록 주기 폴링으로 에디터 등장을 감지해 마운트한다.
- 매치 패턴은 `https://*/admin/shopping/product/detail*` — 아임웹 기본 도메인(*.imweb.me)과 커스텀 도메인 관리자를 모두 커버하며, 해당 경로에서만 실행된다.

## 파일 구조

- `manifest.json` — MV3 선언 (content script, MAIN world)
- `content.js` — 패널 UI + 템플릿 + Froala 연동 (스타일 내장, 외부 의존성 없음)

## 제약

- 요약 설명 필드 전용 (상세 설명 에디터는 대상 아님)
- 적용한 내용은 화면 우측 상단 [저장]을 눌러야 실제 저장된다
- 실측 기준: 아임웹 관리자 상품 편집 화면, Froala 3.1.1 (2026-07 확인)
